# 주간 업무 회고 (2월 3주차)

## CRM 프로젝트
### 1. 주요 업무 성과 (Key Achievements)

이번 주는 설 연휴로 인한 짧은 스프린트(목, 금)였으나, **"FE 사용성 개선(CRM) 및 BE/Infra 안정화"**, **"SEO 대응을 위한 라우팅 로직 개편(홈페이지)"**을 밀도 있게 진행함.

- **UI/UX 전면 개편 및 검색 편의성 고도화 (CRM)**
    
    - **직원 조회 로직 및 UI 리팩토링:** 기존 뷰에서 혼재되어 있던 '직위'와 '담당 업무' 데이터 렌더링을 분리. 직위 데이터가 `null`이거나 `empty`일 경우 '담당 업무'를 렌더링하도록 Fallback(예외 처리) 로직 구현.
        
    - **그리드/리스트 뷰 시각적 가중치 조정:** 리스트 뷰 레이아웃 재배치(이름 > 내선 > 휴대전화 > 직위 > 소속). 주요 식별 데이터인 '내선번호'의 `font-weight` 및 `font-size`를 높여 시각적 접근성 확보.
        
    - **검색 및 정렬 기능 추가:** 직군별(상담/고문/변호사 등) 탭 필터링 및 직위 기반(Index/Rank) 정렬 기능 구현.
        
    - **당사자 필터 다중 선택 및 반응형 UI 적용:**
        
        - 기존 Dropdown 방식의 '유입 경로', '지역지사' 필터를 `FilterChip` 토글 방식으로 변경하여 조작 편의성(Depth 축소) 개선.
            
        - 단일 선택에서 **다중 선택(Multi-select) 지원**으로 확장.
            
        - 모바일 뷰포트(`max-width: 600px`) 대응: 화면 하단에서 올라오는 `Bottom Sheet` 컴포넌트로 필터 UI 분기 처리하여 모바일 사용성 극대화.
            
        - **BE 연동:** 클라이언트에서 다중 선택한 값을 백엔드 `source`, `sendOffice` 파라미터에 쉼표(`,`)로 구분된 문자열(Comma-separated)로 전달하고, 서버에서 이를 파싱하여 쿼리하도록 로직 수정.
            
- **SEO 최적화 및 라우팅 상태 방어 (메인 홈페이지)**
    
    - **클린 URL 구축:** SEO 스코어 하락의 원인이 되던 불필요한 쿼리 파라미터(`ReturnUrl`)를 URL에서 제거.
        
    - **상태 보존(State Preservation) 로직 구현:** 파라미터 제거 후 발생할 수 있는 라우팅 이슈(예: `history.back()` 시 이전 경로 유실)를 방어하기 위해, Web Storage(Session/Local) 기반의 임시 저장소를 활용해 라우팅 상태를 보존 및 복원하는 로직으로 대체.
        
- **기타 실무 운영/데이터 추출**
    
    - 경기북부지사 당사자 상담 건 DB 추출 및 필터링 (기간: '24.11.01 ~ '25.12.19 / `source`: 내선, 메인 홈페이지)
        

### 2. 기술적 도전 및 트러블슈팅 (Technical Challenges & Troubleshooting)

- **검색 장애 디버깅 및 DB 스키마 마이그레이션:** 특정 조건 조합 시 발생하던 검색 API 에러 및 조회 불가 이슈의 원인을 파악하여 핫픽스 진행. 영문 이름, 상세 직급 등 신규 요구사항 데이터 규격에 대응하기 위해 DB 테이블 스키마 확장(Alter Table).
    
- **DB 마이그레이션:** 데이터 유실 및 다운타임 없이 DB 서버 마이그레이션(Migration) 완료.
    
- **데이터 무결성 검증 스크립트 작성:** 마이그레이션 이후 조직 데이터의 정합성(Integrity)을 보장하고 휴먼 에러를 차단하기 위해, 자체 조직 검증 스크립트를 작성하고 파이프라인/배치에 추가하여 시스템 안정성 강화.
    

### 3. 총평 및 향후 과제 (Wrap-up & Next Steps)

> **"이틀짜리 짧은 스프린트, 프론트엔드의 디테일한 UX 튜닝과 백엔드/Infra의 코어 작업을 동시에 쳐낸 밀도 높은 주간"**

단순한 UI 컴포넌트 교체(Dropdown -> FilterChip, Bottom Sheet 적용)를 넘어, 다중 선택에 따른 백엔드 파라미터 핸들링 로직까지 풀스택으로 작업을 진행함. 특히 SEO 개선을 위해 URL 구조를 변경하면서도 UX 저하를 Web Storage 로직으로 선제 방어한 점과, DB 마이그레이션 후 검증 스크립트로 데이터 무결성을 챙긴 점이 좋은 프랙티스였음.
