
# 주간 업무 회고 (2월 4주차)

## 1. 주요 업무 성과 (Key Achievements)

이번 주는 **"CRM 시스템의 핵심 기능(상담/SMS/당사자) 고도화"**, **"메인 홈페이지 SEO 개선"**, 그리고 **"관보 크롤러 자동화 파이프라인 구축"**을 성공적으로 마무리하며 프론트엔드와 백엔드, 인프라를 아우르는 전방위적인 개선을 이뤄냄.

- **CRM 프로젝트: UI/UX 전면 개편 및 기능 고도화**
    
    - **상담 및 당사자 관리 사용성 개선:**
        
        - 상태 체계를 3가지(예약/취소/완료)로 단순화하고, 상태 필터 카드를 4개로 직관적으로 재구성.
            
        - 리스트 테이블 내 드롭다운을 통한 실시간 상태 변경 기능 추가.
            
        - 당사자 신규 등록 시 상담·문자 이력을 동시 추가(초안 모드)하는 일괄 저장 프로세스 최적화 및 당사자 관계(그룹 생성/편집/삭제) 기능 신규 구축.
            
        - 유입경로, 지역지사, 상담일 범위 등을 활용한 다중 필터 기능(FE/BE API 연동) 구현.
            
        - 웹/앱 레이아웃 정렬 및 상세 조회 다이얼로그 팝업 연동(삭제 확인창 추가).
            
    - **SMS 문자 발송 및 예약 시스템 고도화:**
        
        - **인프라/API 전환:** 로컬 저장 방식에서 서버 API 기반으로 전환하여 계정별 양식 관리(CRUD) 체계 구축. 실제 발송 처리를 위해 NCP SENS API 연동.
            
        - **기능 추가:** 예약 문자 추가, 취소, 상세 확인 기능 및 예약 시간 도달 시 자동 발송 처리(전송 완료 시 '성공' 자동 변경) 구현.
            
        - **UI/UX 개선:** 문자 발송 팝업 내 스크롤/다중행 지원, 당사자 검색 드롭다운 적용, 담당자 자동 등록 및 비고란 순차 표시. 문자 이력 자동 업데이트(새로고침 불필요).
            
- **메인 홈페이지: SEO 최적화 및 유지보수**
    
    - **SEO 및 검색 노출 개선:** 지사별 사이트맵 구조 분리 및 관리자 스크래핑 데이터를 기반으로 sitemap을 자동 갱신하는 스크립트 추가.
        
    - **UI 및 정렬 수정:** 지식센터 카테고리명/노출 순서 변경, 변호사 소개 페이지 내 부산지사 담당 변호사 상단 우선 정렬.
        
- **관보 크롤러 자동화 프로젝트 구축**
    
    - 일자별 관보 PDF 전체 자동 다운로드 스크립트 작성 및 Github Actions 스케줄링(평일 13시) 적용.
        
    - 다운로드된 PDF 메일 자동 발송 및 Github 아티팩트 활용 14일 보관 설정.
        

## 2. 기술적 도전 및 트러블슈팅 (Technical Challenges & Troubleshooting)

- **시간대(Timezone) 동기화 이슈 및 상태 표시 오류 해결:** 시스템 시간대 차이로 인해 문자 발송 시간이 실제와 다르게 표기되던 문제(+9시간 오작동)를 한국 시간 기준으로 정확히 매핑하여 해결. 또한, 발송 전인 예약 문자가 '실패'로 노출되던 버그를 고치고 '예약 대기' 상태로 정상 노출되도록 핫픽스 진행.
    
- **필터 버그 및 UI 충돌 픽스:** 필터 적용 버튼 이중 클릭 버그와 배지(Badge) UI로 인한 터치 차단 문제 해결. 상태별 건수와 목록 필터가 불일치하는 현상을 잡아내어 누락되던 전체 건수 집계 오류 수정. 관계인 클릭 시 당사자 상세 화면이 갱신되지 않던 렌더링 이슈 해결.
    
- **DB 마이그레이션 및 파이프라인 안정화:** 기존 복잡했던 상담 상태 데이터(노쇼, 진행중, 빈값 등)를 새 체계에 맞게 마이그레이션하고, 이를 일괄 정리·확인할 수 있는 CLI 스크립트(`migrate:consultation-status`) 작성.
    
- **배포 인프라 스크립트화:** 로컬 DB를 Railway 클라우드로 쉽게 덤프/임포트 할 수 있는 스크립트(`deploy-db-to-railway.js`)를 구축. 앱 배포 시 SMS 템플릿 관련 DB 마이그레이션이 자동 실행되도록 하여 다운타임과 휴먼 에러 차단.
    

## 3. 총평 및 향후 과제 (Wrap-up & Next Steps)

> **"사용자 편의를 위한 프론트엔드 디테일 튜닝부터, 백엔드 API 연동, DB 마이그레이션 및 자동화 파이프라인 구축까지 풀스택 역량이 돋보인 밀도 높은 주간"**

단순한 기능 추가를 넘어 당사자 관계 설정, 실시간 상태 업데이트, 다중 필터링 등 사용자(CRM 실무자)의 편의성을 극대화하는 데 집중함. 특히 Timezone 버그 해결과 상태값 매핑을 통해 데이터의 정확도를 높이고, 배포/마이그레이션 과정을 스크립트화하여 인프라 안정성을 확보한 점이 매우 훌륭한 프랙티스였음.

**향후 과제:**

- **크롤러 고도화:** 관보 크롤러를 구글 클라우드 환경(GCP Storage)과 연동하여 다운로드된 PDF 파일이 클라우드에 자동 저장되도록 파이프라인 확장.
    
- **홈페이지 안정화:** 변호사 소개 페이지 내 목록 및 검색 화면에서 발생하는 잔여 오류 확인 및 픽스 진행.
