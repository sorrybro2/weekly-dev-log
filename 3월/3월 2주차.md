# 주간 업무 회고 (3월 2주차)

## 1. 주요 업무 성과 (Key Achievements)

CRM 시스템의 메시징 및 고객 관리 기능 고도화와 구글 워크스페이스 연동을 통한 인프라 자동화 기반 마련

- **CRM 프로젝트: 발송 시스템 고도화 및 구글 연동 서비스 구축**
    
    - **UI/UX 개선 및 프론트엔드:**
        
        - SMS 첨부파일 드래그 앤 드롭 지원 및 상세 다이얼로그 내부 스크롤 추가를 통한 사용성 개선
            
        - 당사자 검색형 드롭다운 적용 및 예약 달력 내 윤년(2월 29일) 반영 고도화
            
        - User Selector 및 Datepicker 컴포넌트 신규 구성
            
    - **메시징(SMS) 고도화 (Backend/Frontend):**
        
        - 문자 발송 폼 내 제목 입력란 추가 및 실제 발송 형태 기반의 반응형 미리보기 UI 개편
            
        - NCP 과금 기준을 반영한 정확한 바이트(Byte) 수치 표기 및 발송 상태(수신 코드, 실패 여부) 노출 UI 개편
            
    - **구글 서비스 연동 및 데이터베이스:**
        
        - 구글 OAuth 토큰 및 연락처 동기화 상태 관리를 위한 신규 테이블 2종 생성 및 마이그레이션 스크립트 작성
            
        - Google People API 기반의 연락처 수동/자동 동기화 파이프라인 구축 및 다중 계정 선택 기능 구현
            
        - 사내 DB와 Google Admin 커스텀 스키마(`emp_info`) 간 직원 정보 양방향 동기화(Push/Pull) API 신설
            
    - **인프라 설정 및 문서화:**
        
        - 백엔드 컨테이너 시작 명령어를 `npm start`로 통합하고 Cloud Run 및 Cloud SQL 환경에 맞춘 배포 가이드 전면 개편
            
        - 첨부 이미지 용량 제한을 환경변수(ENV)로 분리하여 관리 효율성 증대
            
- **법승 메인 홈페이지 유지보수: 편의성 개선 및 SEO 자동화**
    
    - **UI/UX 및 폼 개선:**
        
        - 상담 신청 폼 내 이름 및 전화번호 입력 시 브라우저 자동완성(History) 기능 비활성화 일괄 적용
            
        - 모바일 환경 통화 링크 및 CKEditor 내 전화 연결(`tel:`) 프로토콜 삽입 오류 수정
            
    - **SEO 최적화 및 인프라:**
        
        - Playwright를 도입하여 전국 10개 지점의 사이트맵(XML) 크롤링 및 갱신 자동화 스크립트 구축
            
        - `robots.txt` 내 지역별 사이트맵 등록 및 Git 추적 제외 파일(`.claude` 등) 환경 정비
            

## 2. 기술적 도전 및 트러블슈팅 (Technical Challenges & Troubleshooting)

- **NCP 400 에러 및 발송 타입 판정 로직 개선:** 단순 글자 수 기준으로 발생하던 발송 오류를 방지하기 위해 실제 본문 바이트(Byte) 기준으로 SMS/LMS 타입을 자동 판정하도록 서버 로직을 보정함. 추가로 NCP 400 에러 발생 시 원인 노출을 강화하고 광고(AD) 문구가 자동 처리되도록 백엔드 로직을 수정함.
    
- **구글 API 연동 권한 및 예외 처리 세분화:** Google 로그인 시 연락처 권한(Contacts Scope)을 획득하는 과정에서 발생할 수 있는 오류(권한 부족, 토큰 만료, API 미활성화 등)를 명확히 구분함. 이를 통해 사용자 안내 메시지를 세분화하고 서버 측 오류 로그를 강화하여 시스템 안정성을 확보함.
    
- **직원 마스터 데이터 정합성 보장:** 파편화된 직원 정보를 일원화하기 위해 CSV 기반의 마스터 데이터 자동 갱신(Upsert) 스크립트를 개발함. 또한, 구글 디렉터리와 사내 Cloud SQL 간의 데이터 동기화를 위해 전체 사용자의 JSON 덤프 추출 및 조직/업무 매핑 테이블 동기화 로직을 성공적으로 구축함.
    

## 3. 총평 및 향후 과제 (Wrap-up & Next Steps)

> **"CRM 메시징 정확도 향상과 구글 생태계 연동을 통해 데이터 동기화의 토대를 다진 한 주"**

이번 주에는 바이트 기반의 정교한 메시징 시스템을 구축하여 발송 정확도를 크게 높였으며, 구글 연락처 및 디렉터리 API를 활용한 양방향 데이터 동기화 파이프라인을 마련했습니다. 더불어 홈페이지 사이트맵 갱신을 자동화하여 유지보수 리소스를 절감하고 검색엔진 최적화(SEO)를 한층 강화하는 기술적 성과를 달성했습니다.

**향후 과제:**

- **구글 연락처 동기화 통합 테스트:** 구현 완료된 당사자 정보 및 구글 연락처(People API) 연동 로직에 대한 실환경 통합 테스트 진행
    
- **Google Workspace 양방향 API 연동 테스트:** 신설된 관리자 정보 동기화 엔드포인트(`/admin-emp-info/push, pull` 등)의 데이터 정합성 검증 및 실서버 적용
