## 2. 기술적 도전 및 트러블슈팅 (Technical Challenges & Troubleshooting)

- **Cloud SQL 연결 오류(EAI_AGAIN) 해결:** DB 연결 로직을 기존 host 기반에서 socketPath 방식으로 변경하고, SSL 및 포트 분기 처리를 보완하여 클라우드 환경의 연결 안정성을 확보함.
    
- **OAuth 인증 예외 처리 강화:** OAuth 로그인 시 발생하는 `invalid_grant` 오류를 완화하기 위해, 인증 코드 정규화(공백 제거 및 `+` 변환) 로직을 추가하여 로그인 실패율을 개선함.
    
- **파일 업로드 안정성 확보:** 다양한 파일 환경을 지원하기 위해 octet-stream 형식 업로드를 허용하고, GCS 인증 폴백 및 이미지 프록시 URL 반환 로직을 추가하여 MMS 파일 첨부의 안정성을 높임.
    
- **데이터 정합성 및 예외 처리 고도화:** 의뢰인 정보 저장 시 발생하던 생년월일 날짜 오류와 부재 상태의 취소 분류 누락 문제를 수정함. 또한 404 오류 시 백엔드 상세 로그를 남기고, 프론트엔드에서는 사용자 친화적 메시지를 파싱하도록 개선함.
