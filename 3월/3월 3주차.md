# 주간 업무 회고 (3월 3주차)

## 1. 주요 업무 성과 (Key Achievements)

CRM 시스템의 백엔드 데이터 동기화 파이프라인 자동화 및 프론트엔드 사용성 고도화를 달성함.

- **CRM 프로젝트: Google Admin 연동 데이터 동기화 및 관리 시스템 고도화**
    
    - **백엔드/데이터 동기화 (BE/DB):**
        
        - Google Admin ↔ DB 간 직원 정보 양방향 동기화 서비스(Push/Pull) 및 웹훅 엔드포인트 신규 구축
            
        - CSV 기반 마스터 데이터 동기화 로직 구현 및 Google Admin 맞춤형 스키마 자동 Push 스크립트 추가
            
        - 직원 식별 기준을 고유 값인 '이메일'로 고정하고, 이메일 별칭(Alias) 및 직무 키(duty_key) 컬럼을 신규 추가하여 데이터 매칭 정확도 향상
            
        - 데이터 시드(Seed) 진행률 로그 및 누락 통계 추적 기능 도입
            
    - **프론트엔드 및 UI/UX (FE/UI):**
        
        - 다중 직원 연락처 동기화 다이얼로그(ContactSyncDialog) 신규 개발 및 로그인 사용자 기준 공용 계정 자동 선택 로직 적용
            
        - 상담 관리 화면에 공통 카드 컨테이너를 적용하고, 모바일 환경에 맞춰 헤더 및 버튼 배치 전면 리팩토링
            
        - 검색 당사자(담당자) 필터 상태 세션 저장 로직 추가로 페이지 새로고침 시 필터 초기화 방지
            
        - 당사자 관리 화면 진입 시 전화번호 자동 입력 기능 구현 및 화면 네비게이션 동선 수정
            
    - **인프라 및 시스템 공통 (Infra/System):**
        
        - 범용 '고급 필터 위젯' 도입 및 조직 노출 기준에 따른 메타데이터 마이그레이션 완료
            
        - 클라우드 SQL 운영 환경 및 신규 동기화 옵션에 대한 프로젝트 README 문서 최신화
            
- **법승 메인 홈페이지: 폼 데이터 전송 로직 개선 및 UI 최적화**
    
    - **프론트엔드 로직 고도화:**
        
        - 회생파산센터 문의 및 메인 페이지 폼에서 사건 영역 선택 시 지역 데이터가 '서울'로 자동 지정되도록 전송 로직 개선
            
        - 조건문 비교 기준을 한글명에서 컨트롤러 영문명으로 변경하여 시스템 안정성 확보
            
    - **UI/UX 수정:**
        
        - 푸터(Footer) 영역의 불필요한 지역별 사무소 드롭박스 및 중국어 문의 이메일 노출 임시 숨김(주석) 처리
            

## 2. 기술적 도전 및 트러블슈팅 (Technical Challenges & Troubleshooting)

- **동시성 제어 및 데이터 중복 방지:** Cloud Run 다중 인스턴스 환경에서 자동 동기화 스케줄러의 중복 실행을 막기 위해 DB 락(Lock)을 적용함. 또한, 이름 변경 시 발생하던 중복 데이터 생성 문제를 고유 식별자(이메일) 매칭으로 원천 차단함.
    
- **데이터 무결성 보장 및 쿼리 최적화:** MySQL 조회 시 발생하던 `LIKE ESCAPE` 구문 파싱 오류 및 정렬 오류를 수정하고 조직 간 교집합 비교 방식을 적용함. 조직 및 직원 부서 테이블의 외래키(FK) 제약조건을 보강하고 서버 구동 시 자동 스키마 보강 스크립트를 적용함.
    
- **API 응답 호환성 및 비동기 상태 관리:** API 응답 키 형식(snake_case vs camelCase) 차이로 인한 공용 연락처 목록 누락 버그를 보정함. 또한, 비동기 요청 시 이전 응답 데이터가 현재 화면 필터를 덮어쓰는 타이밍 이슈를 해결하여 데이터 표출 안정성을 확보함.
    
- **구글 OAuth 인증 보안 및 예외 처리 강화:** 최소 권한 원칙을 적용해 불필요한 Google Contacts 권한(Scope)을 제거하고, 레거시 액세스 토큰 저장 로직을 삭제하여 보안 구조를 간소화함. 공용 계정 미매칭 시에도 에러 대신 다이얼로그가 정상적으로 열리도록 예외 처리를 보완함.
    

## 3. 총평 및 향후 과제 (Wrap-up & Next Steps)

> **"데이터 정합성을 보장하는 견고한 동기화 파이프라인 구축과 모바일 친화적 UI 개편으로, 시스템 안정성과 실무자의 업무 효율을 동시에 끌어올린 한 주"**

이번 주에는 Google Admin과 사내 DB 간의 양방향 데이터 동기화 인프라를 성공적으로 안착시키고 다중 연락처 동기화 기능을 고도화하여 데이터 관리의 편의성을 대폭 향상했습니다. 더불어 인증 로직 간소화와 쿼리 최적화를 통해 백엔드 안정성을 확보했으며, 사용자 피드백을 반영한 세션 기반 필터 유지 및 모바일 UI 리팩토링으로 향상된 사용자 경험(UX)을 제공했습니다.

**향후 과제:**

- **AI 기반 업무 파이프라인 도입 검토:** 구글 클라우드 컨퍼런스에서 학습한 '바이브 코딩(Vibe Coding)' 및 Agentic 플랫폼(Gemini Enterprise 등)을 사내 개발/운영 파이프라인(SDLC) 및 업무 흐름 자동화에 적용할 수 있는 구체적 방안 검토.
    
- **임시 조치된 UI 영역의 정식 개편:** 현재 임시로 숨김(주석) 처리된 법승 홈페이지 푸터 드롭박스 및 CRM 프론트엔드의 불필요한 직원 등록 버튼 등에 대한 최종 요구사항 확정 및 정식 코드 정리.
