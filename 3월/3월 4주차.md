# 주간 업무 회고 (3월 4주차)

## 2. 기술적 도전 및 트러블슈팅 (Technical Challenges & Troubleshooting)

- **작성자 식별 체계(인증) 고도화 및 하위 호환성 보장:** 기존 이름/이메일(`gws_id`) 기반의 데이터 식별 기준을 고유 식별자인 `employee_key`로 전면 통일함. 이 과정에서 로그인 응답, 로컬 세션, JWT에 키를 저장하도록 로직을 개선했으며, 기존 레거시 데이터 및 재로그인 이전 세션에서도 시스템이 끊김 없이 작동하도록 하위 호환성(Fallback) 처리를 보강하여 데이터 정합성을 확보함.
    
- **NCP SMS API 발신 지역 조회 성능 병목 해결:** 문자 발송 내역 조회 시 응답받은 지역번호(051, 042 등)를 분석하는 과정에서 불필요한 API 재호출이 반복되는 현상을 파악함. 이를 해결하기 위해 발신 지역 조회 결과에 캐싱(Caching) 시스템을 도입하여 데이터 매핑 방식을 개선하고 전체적인 화면 로딩 성능을 최적화함.
    
- **Google Admin 데이터 동기화 예외 처리 및 안정화:** Admin 계정 동기화 시 연락처가 삭제될 경우 내부 DB에 빈 값(NULL)으로 정상 반영되도록 공통 로직을 수정함. 또한, 테스트용 보조 이메일(`*.test-google-a.com`) 유입을 차단하는 필터링 규칙을 적용하고, 생성된 임시 직원 키가 실제 키로 안전하게 전환되도록 매핑 로직을 보완하여 동기화 안정성을 크게 높임.
