## 2. 기술적 도전 및 트러블슈팅 (Technical Challenges & Troubleshooting)

- **[Timezone(UTC/KST) 데이터 정합성 이슈 해결]:** 당사자/상담 생성 및 문자 발송 시각, 구글 시트 연동 과정에서 시스템 시간이 UTC로 이중 해석되어 9시간 오차가 발생하는 문제를 발견함. 백엔드 타임존을 한국 시간(Asia/Seoul)으로 고정하고, 프론트엔드 데이터 파싱 로직을 안정화하여 전 영역에서 일관된 로컬 시각이 노출되도록 조치함.
    
- **[DB 조회 성능 저하 및 무한 로딩 최적화]:** 복잡한 8중 JOIN으로 인해 당사자 목록 조회 속도가 크게 저하됨. 서브쿼리를 LEFT JOIN으로 리팩토링하고, 상태 및 등록일 등 주요 조건에 DB 인덱스를 추가함. 또한 데이터 조회와 카운트 쿼리를 병렬 처리(Promise.all)하여 화면 대기 시간을 대폭 단축함.
    
- **[레거시 구글 시트 연동 구조 개편]:** 기존 Apps Script 기반의 느린 동기화를 대체하기 위해 DB 웹훅(Webhook) 엔드포인트를 신규 구축함. 시트 수정 시 DB에 즉시 반영되도록 개선했으며, Cloud Run을 활용해 DB에서 데이터 삭제 시 구글 시트의 해당 행(Row)도 일괄 삭제되도록 양방향 동기화 무결성을 확보함.
    
- **[API 예외 처리 및 안정성 확보]:** 문자 내역 조회 시 발생하던 500 서버 에러 구조를 개선하여 빈 값 없이 응답되도록 수정하고, 당사자 삭제 API의 DB 커넥션 누수 문제를 해결하여 시스템 안정성을 높임.
