# 주간 업무 회고 (4월 1주차)

## 1. 주요 업무 성과 (Key Achievements)

대규모 데이터 데이터베이스(DB) 마이그레이션 성공 및 내/외부 연동 파이프라인 고도화를 통한 시스템 안정성 확보

- **법승 CRM 프로젝트: 대규모 데이터 이관 및 프론트엔드/백엔드 인프라 고도화**
    
    - **UI/UX 및 프론트엔드 개선:**
        
        - 당사자 조회 및 검색 모달 도입, 연락처 필터(사무실/유입경로/상담유형)를 카드형으로 직관적으로 개편하여 사용자 동선 최적화함.
            
        - 텍스트/태그 겹침, 긴 텍스트 말줄임, 데이터 테이블 4자리 숫자 페이지네이션 개행 버그 등 뷰(View) 렌더링 최적화함.
            
        - 서비스 브랜딩 통일을 위해 PWA 앱 이름, 브라우저 탭, 로고 이미지를 '법승 CRM'으로 일괄 교체 및 반응형 처리함.
            
    - **데이터베이스 및 백엔드/인프라 연동:**
        
        - Google Sheets(CSV)의 데이터를 Cloud SQL로 마이그레이션하는 스크립트 구축 및 1차 적재 완료함 (당사자, 상담, 문자/전화 등 약 9.8만 건).
            
        - 구글 디렉터리(Google Directory) 관리자 스키마와 내부 DB(employee_key 유지) 간의 단건/배치 동기화 API 파이프라인 연동함.
            
        - 당사자 등록 시 KST(한국 시간) 기준 타임스탬프를 적용하여 구글 시트와 양방향 동기화되도록 로직 보완함.
            
- **법승 메인 홈페이지: 검색 엔진 최적화(SEO) 및 기술 리스크 기반 외주 비용 방어**
    
    - **콘텐츠 및 SEO 최적화:**
        
        - 다국어(베트남어/중국어) 및 주요 지사 FAQ를 단일 JSON-LD 노드로 통합하여 SEO 검색 노출도 및 관리 효율성을 극대화함.
            
        - 부산 지사 신규 변호사 프로필 추가, 주요 5개 지사 최신 정보 갱신, 메인 화면 지표 기준(지사 사무실) 변경 적용함.
            
    - **기술 검토 및 리스크 관리:**
        
        - 외주 개발 건에 대해 자체 소스 코드 분석 결과, DB 권한 부재로 인한 라이브 서버 장애 리스크를 사전 식별하고 무리한 배포를 차단함.
            

## 2. 기술적 도전 및 트러블슈팅 (Technical Challenges & Troubleshooting)

- **외부 연동 API 권한 분리 및 안정성 확보:** 기존 구글 시트 직접 연동 시 발생하던 권한 충돌 이슈를 해결하기 위해 전용 Cloud Run 서비스로 로직을 분리함. 또한 전용 서비스 장애 시 시스템이 멈추지 않도록 기존 방식을 안전장치(Fallback)로 유지하는 아키텍처를 적용함.
    
- **문자 조회 응답 지연(성능 최적화):** 목록 조회 시 불필요하게 호출되던 외부 API(NCP)와 복잡한 서브쿼리를 제거하여 렌더링 성능을 대폭 개선함. 이와 함께 소속 지사를 기준으로 권한 필터링 로직을 정교화하여 데이터 보안을 강화함.
    
- **데이터 정합성 보장 및 검색 예외 차단:** 기존 이름 기반 검색 예외 처리 로직으로 인해 잘못된 정보가 노출되는 문제를 원천 차단하고자 partyKey 필터를 강제 적용함. 연락처 동기화 시에도 퇴직자 계정은 제외되도록 쿼리를 최적화함.
    
- **DB 스키마 문서화 자동화:** 데이터베이스 구조 현행화를 위해 최신 Cloud SQL 스키마를 읽어 외래키(FK) 기반의 물리/논리 ERD(Mermaid 형식)를 자동 생성하는 기능을 세팅함.
    
- **비정형 데이터 마이그레이션 및 포맷 예외 처리:** 셀 내부 줄바꿈 등 복잡한 데이터 파싱을 지원하는 스크립트를 작성하고, 연락처 컬럼 타입을 TEXT로 변경하여 20자 길이 제한 문제를 해결함. 또한 5자리 미만 연락처 저장 시 예외 로그를 남기는 등 검증 로직을 고도화함.
    

## 3. 총평 및 향후 과제 (Wrap-up & Next Steps)

> **"복잡한 외부 데이터의 성공적인 내재화 및 선제적 리스크 방어를 통해 개발팀의 기술 주도권을 확립한 한 주"**

구글 기반의 파편화된 데이터를 Cloud SQL 중심의 시스템으로 성공적으로 통합·이관하였으며, 동기화 프로세스 분리를 통해 인프라 안정성을 확보했습니다. 또한, 서드파티 외주 건에 대한 정확한 코드/인프라 분석으로 치명적 서버 장애 리스크를 방어하고 회사 차원의 비용 절감을 이끌어냈습니다.

**향후 과제:**

- **구글 시트 연동 403 권한 오류 해결:** API 최종 쓰기 단계에서 발생하는 권한 오류에 대해 런타임 자격증명(client_email) 로그를 분석하고 서비스 계정 권한 재설정 작업 진행.
    
- **최소 범위 역제안을 통한 외주 협상:** 프론트엔드 자체 구현을 담보로, 외주사에 "필수 DB 컬럼 1개 및 API 파라미터 오픈"만 요구하는 비용 절감 협상 타결.
    
- **직원 정보 동기화 고도화:** 사전 구축된 직위명 정규화 매칭 유틸리티를 활용하여 구글 직원 동기화 본 개발 마무리.
    
- **Cloud SQL 메타데이터 조회 검증:** 라이브 DB 메타데이터 조회 제한 이슈에 대한 원인 파악 후 추가 검증 완료 및 ERD 현행화 반영.
