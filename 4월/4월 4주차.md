# 주간 업무 회고 (4월 4주차)

## 1. 주요 업무 성과 (Key Achievements)

CRM 시스템의 안정적인 베타테스트 환경 구축 및 기존 홈페이지의 인프라 최적화, 3차 리뉴얼 아키텍처 설계 완료

- **CRM 시스템 (베타테스트): UI/UX 고도화 및 베타 환경 안정화**
    
    - **UI/UX 및 프론트엔드 개선:**
        
        - 캘린더 미리보기 내 상담사 직위 표기 제거 및 축약 로직 공통 함수 모듈화 적용
            
        - 문자 양식 목록 가로 스크롤 추가 및 공간 패딩·컨트롤러 최적화
            
        - '회생 관리' 신규 메뉴 라우트 추가 및 상태 필터·요약 카드를 포함한 인입 목록 화면 신규 구축
            
        - 회생 질문지 기반 신규 입력 폼 개발, 스크립트 패널 연동 및 레이아웃(독립 스크롤) 개선
            
        - 회생 화면 내 주요 용어를 '상담' 기준으로 전면 통일하여 UI 일관성 확보
            
        - 신규 상담 카드에 임시 문구 대신 실제 진행 상태(접수, 예약, 취소, 완료) 직관적 노출 적용
            
    - **문서화 및 인프라 설정:**
        
        - 베타테스트 전용 도메인(crm.lawwin.co.kr) 매핑 및 적용 완료
            
        - 최신 개발 사양(동기화 다이어그램 교체, 마이그레이션, 비활성 API 등)에 맞춘 README 전면 최신화
            
        - 실무자의 원활한 시스템 적응을 돕는 사용자 매뉴얼 작성 및 2차 내용 보강 완료
            
- **법승 메인 홈페이지: 인프라 환경 개선 및 검색 엔진 최적화(SEO)**
    
    - **빌드 및 백엔드 최적화:**
        
        - 배포 시 산출물 폴더(publish_output) 중복 포함 방지 규칙 추가 및 불필요한 산출물 정리
            
        - URL별 SEO 문구 업로드 및 조회를 위한 신규 API 개발
            
        - URL 쿼리스트링 및 후행 슬래시 정규화를 통한 일관된 SEO 문구 매칭 환경 구축
            
        - 업로드된 SEO 데이터 메모리 캐싱 처리를 통한 조회 성능 대폭 향상
            
- **법무법인 법승 홈페이지 리뉴얼 (3차): 차세대 아키텍처 및 기획안 수립**
    
    - **기획 및 기술 스택 설계:**
        
        - 100% 내재화 및 웹 성능 지표(Core Web Vitals) 개선을 위한 기술 스택(Next.js, NestJS, PostgreSQL) 선정
            
        - 관리자 페이지 보안 강화를 위한 Google SSO 및 다중 보안 레이어(Cloudflare → GCP LB → GCP IAP → Cloud Run) 설계
            
        - AEO(AI 검색 최적화) 대응을 위한 Schema.org 구조화 데이터 자동 삽입 구상
            
        - 유사 판례 검색 등 비정형 데이터 벡터 검색을 위한 PostgreSQL(pgvector) 도입 리서치 적용
            
    - **일정 및 예산 산출:**
        
        - 개발 버퍼 1개월을 포함한 총 7개월(2026. 08 ~ 2027. 02) 로드맵 초안 수립
            
        - 클라우드 인프라(GCP) 월별 유지 예산 및 프리랜서 UI/UX 디자이너 인건비 예산 산출 완료
            

## 2. 기술적 도전 및 트러블슈팅 (Technical Challenges & Troubleshooting)

- **비동기 조회 지연에 따른 관계인 데이터 덮어쓰기 오류 해결:** 당사자 정보 수정 화면에서 최근 편집한 데이터가 덮어씌워지는 현상을 방지하기 위해, 접근 권한 우선 확인 후 목록 전체를 그룹 단위로 새로 불러오도록 데이터 조회 로직을 전면 개편함. 또한 상세 페이지 복귀 시 관련 데이터만 부분 새로고침 처리하여 렌더링 속도를 대폭 개선함.
    
- **상담 편집 화면 내 일정 접두사 누락 버그 보완:** 데이터 조회 시 상담 테이블에 접두사가 누락된 경우 일정 테이블의 값을 대체하여 반환하도록 예외 처리를 보완하고, 신규 상담 저장 시 접두사가 양측 테이블에 정상적으로 연동 및 저장되도록 데이터 정합성을 확보함.
    
- **메인 홈페이지 접속 장애(500 에러군) 근본 원인 규명:** 애플리케이션 코드 결함이 아닌 배포 서버 환경의 문제로 파악함. 서버 디스크 용량 부족으로 인해 호스팅 번들(DLL) 설치가 누락되어 500.30/500.31 에러가 발생했으며, 500 에러는 설정 파일(appsettings.json) 내 DB 주소 매핑 오류가 원인임을 확인함.
