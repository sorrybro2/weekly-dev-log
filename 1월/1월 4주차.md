# 📅 주간 업무 회고(1월 4주차)

## 1. 이번 주 요약 (Executive Summary)

이번 주는 홈페이지의 **뉴스 섹션 전면 리뉴얼**과 **글로벌(중국/베트남) 페이지 접근성 강화**를 통해 사용자 경험을 개선했습니다. CRM 프로젝트의 경우, 기존 데이터 이관 계획을 잠정 보류하고 **모바일 앱 환경에서의 Google 로그인 인증 시스템 구축**에 역량을 집중하여 기술적 난이도가 높은 네이티브 인증 연동을 성공적으로 완료했습니다.

---

## 2. 프로젝트별 상세 성과

### 🅰️ 메인 홈페이지 (Global & UI/UX)

뉴스 섹션의 시각적 개선과 다국어 지원을 위한 기반 작업을 수행했습니다.

- **UI/UX 리뉴얼 (Renewal):**
    
    - **뉴스 섹션:** UI 전면 개편 및 카테고리별 '최신순' 파싱 로직 구현. 저해상도(높이 700px) 화면 대응을 위한 여백(UI) 최적화.
        
    - **미디어 섹션:** 모바일 레이아웃 및 썸네일 스타일링 개선.
        
- **글로벌 기능 확장 (Global Expansion):**
    
    - **지역 선택:** 메인 및 헤더 드롭다운에 중국/베트남 옵션 추가 및 자연스러운 페이지 이동 로직 구현.
        
    - **중국 페이지:** 성공 사례 확인 버튼 추가 및 반응형 디자인 적용.
        
    - **번역 기능:** EN, CN, JP, VN 번역 기능 탑재 (유료 결제 후 UI 노출 예정).
        
- **유지보수:** 고문/전문위원 메뉴 탭 누락 현상 수정 및 복구.
    

### 🅱️ CRM APP (Mobile Authentication)

웹(Web) 중심에서 앱(App) 환경으로 인증 시스템을 확장했습니다.

- **전략 변경:**
    
    - 초기 계획했던 'Google Sheet $\to$ SQL 데이터 이관'은 보류하고, **모바일 로그인 환경 구축**을 우선순위로 변경하여 진행.
        
- **개발 환경 업그레이드:**
    
    - Android 빌드 환경 최적화 (Java 17, AGP 8.2.1 적용).
        
    - 패키지명 정리 및 MainActivity 경로 수정.
        
- **기능 구현 (Native Login):**
    
    - **Google Sign-In:** `google_sign_in` 패키지를 활용한 네이티브 로그인 구현.
        
    - **인증 연동:** Firebase 프로젝트 연동, OAuth Client ID 설정, SHA-1 인증서 등록.
        
    - **토큰 교환:** `serverClientId` 설정을 통해 백엔드에서 검증 가능한 인증 토큰 획득 로직 완성.

## 3. 기술적 이슈 및 해결 (Troubleshooting)

|**구분**|**이슈 항목**|**원인 분석 및 해결**|
|---|---|---|
|**UI/UX**|**저해상도 화면 깨짐**|**원인:** 고정된 여백 값으로 인해 높이 700px 이하 화면에서 요소 겹침<br><br>  <br><br>**해결:** 미디어 쿼리를 활용한 상하 여백 동적 조정으로 대응|
|**Auth**|**모바일 OAuth 연동**|**원인:** 웹과 다른 모바일 네이티브 환경(Android)의 인증 흐름 차이<br><br>  <br><br>**해결:** Firebase 콘솔 내 SHA-1 지문 등록 및 `google-services.json` 재구성으로 인증 통로 확보|
|**Build**|**Android 빌드 오류**|**원인:** 최신 라이브러리와 구버전 AGP(Android Gradle Plugin) 호환성 문제<br><br>  <br><br>**해결:** Java 17 및 AGP 8.2.1로 업그레이드하여 빌드 파이프라인 최신화|
